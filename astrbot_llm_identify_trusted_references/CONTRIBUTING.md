# Contributing Trusted Reference Candidates

## Requirements

- The audit must target an official provider endpoint, not a proxy, mirror, relay, or reseller domain.
- The package must use schema `trusted-reference-candidate/v1`.
- The package must be generated by the plugin's voluntary contribution export or match the schema exactly.
- Raw prompts, raw completions, API keys, headers, account IDs, IPs, and private content must be absent.

## Review Flow

1. Submit a candidate through an issue or pull request.
2. Maintainers check endpoint class, schema validity, privacy constraints, and multi-run consistency.
3. Accepted records are normalized into `data/accepted/`; rejected or inconclusive records remain as review artifacts only.

## Naming

Use:

```text
data/candidates/<provider>/<model>/<created_at_bucket_or_hash>.json
```
